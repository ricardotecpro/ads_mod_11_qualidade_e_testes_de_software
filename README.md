# 🧪 Qualidade e Testes de Software

Este repositório contém o material didático completo para o curso de **Qualidade e Testes de Software**.

## 🎯 Objetivo Geral

Capacitar o aluno a planejar, executar e documentar testes de software, aplicando melhores práticas de programação, TDD, modelos de qualidade e ferramentas de apoio, garantindo a conformidade entre especificação e produto final.

## 📚 Estrutura do Curso (16 Aulas)

O curso está dividido em módulos estratégicos que cobrem todo o ciclo de vida de testes:

### 🏛️ Fundamentos e Modelos
*   **Aula 01**: Qualidade de Software e Papel do QA
*   **Aula 02**: Modelos de Qualidade (CMMI, MPS.br)
*   **Aula 03**: Ciclo de Vida de Testes (STLC)
*   **Aula 04**: Documentação e Controle de Testes

### 💻 Desenvolvimento e Boas Práticas
*   **Aula 05**: Melhores Práticas de Programação (Clean Code)
*   **Aula 06**: Controle de Versão e Integração Contínua (CI)
*   **Aula 07**: Técnicas de Teste – Caixa Preta
*   **Aula 08**: Técnicas de Teste – Caixa Branca

### 🧪 Tipos de Teste e TDD
*   **Aula 09**: Tipos de Testes Funcionais
*   **Aula 10**: Testes Não Funcionais e Usabilidade
*   **Aula 11**: Desenvolvimento Orientado por Testes (TDD)
*   **Aula 12**: Testes Unitários e de Integração

### 🤖 Automação e Ferramentas
*   **Aula 13**: Automação de Testes Web (Selenium/Playwright)
*   **Aula 14**: Testes de API (Postman)
*   **Aula 15**: Gestão de Defeitos e Ferramentas de Apoio (Jira)
*   **Aula 16**: Projeto Integrador Final

---

## 🛠️ Tecnologias e Ferramentas
*   **Testes Unitários**: JUnit / PyTest
*   **Automação Web**: Selenium / Playwright
*   **API**: Postman / RestAssured
*   **DevOps**: GitHub Actions, CI/CD
*   **Documentação**: Mermaid.js, Termynal.js

## 🚀 Como Executar Localmente
Este projeto utiliza [MkDocs](https://www.mkdocs.org/) com o tema [Material](https://squidfunk.github.io/mkdocs-material/).

1.  **Instale as dependências**:
    ```bash
    pip install -r requirements.txt
    ```
2.  **Rode o servidor local**:
    ```bash
    python -m mkdocs serve
    ```
3.  Acesse `http://localhost:8000` no seu navegador.
