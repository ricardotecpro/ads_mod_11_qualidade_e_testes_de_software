# Exercícios: Aula 12 - Unitários e Integração 🔗

## 🟢 Básico

1.  **Pirâmide de Testes**: Por que os testes unitários devem estar na base da pirâmide (em maior quantidade) e os testes de UI no topo?
2.  **Padrão AAA**: O que deve ser feito em cada uma das etapas: **Arrange**, **Act** e **Assert**?

## 🟡 Intermediário

3.  **Mocks**: Explique o conceito de um "DB Mock". Por que ele é fundamental para garantir que um teste unitário seja rápido e determinístico?
4.  **Integração**: Qual o principal risco de ter 100% de cobertura de testes unitários, mas nenhum teste de integração no projeto?

## 🔴 Desafio

5.  **Refatoração para Teste**: Você tem um método que faz um cálculo complexo e, ao final, envia um log para um servidor externo via HTTP. Como você reformularia esse método para que o cálculo possa ser testado unitariamente sem depender da internet?
