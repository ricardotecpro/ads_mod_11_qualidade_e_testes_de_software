# Projeto 15: Gestor de Crises 🐛

## 🎯 Objetivo
Priorizar e gerenciar um backlog de defeitos críticos.

## 🛠️ O que fazer
Backlog de Bugs encontrados 2 dias antes do lançamento:
- Bug A: Botão de ajuda com link quebrado (Severidade Baixa).
- Bug B: Usuário consegue comprar sem ter saldo (Severidade Crítica).
- Bug C: Site demora 10 segundos para abrir no iPhone (Severidade Média).

1.  Defina a **Prioridade** de correção (1, 2, 3) para cada um.
2.  Escreva um argumento para o PO (Dono do Produto) sobre por que o Bug B impede o lançamento do sistema.
3.  Desenhe o ciclo de vida que esse Bug B percorrerá desde a abertura até o fechamento.

## 📤 Entrega
Plano de ação para os bugs e justificativa técnica da priorização.
