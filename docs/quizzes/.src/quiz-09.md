# Quiz 09 - Tipos de Testes Funcionais ✅

1. Qual o principal objetivo do Teste de Regressão?
   - [ ] Encontrar novos bugs em funcionalidades novas.
   - [x] Garantir que alterações recentes não quebraram funcionalidades que já estavam estáveis.
   - [ ] Deletar o código antigo.
   - [ ] Testar a velocidade da rede.
   > **Explicação**: Regressão foca na manutenção da estabilidade do sistema após mudanças.

2. O que é um "Smoke Test" (Teste de Fumaça)?
   - [ ] Um teste feito enquanto o servidor está pegando fogo.
   - [x] Um conjunto básico de testes para verificar se as funções essenciais do sistema operam minimamente.
   - [ ] Um teste de segurança contra hackers.
   - [ ] O teste final antes de aposentar o software.
   > **Explicação**: Serve para validar se o build é minimamente estável para seguir para testes mais profundos.

3. O Teste de Aceitação do Usuário (UAT) é realizado por:
   - [ ] Pelo desenvolvedor que escreveu o código.
   - [ ] Pelo compilador.
   - [x] Pelos usuários finais ou clientes do software.
   - [ ] Por robôs de inteligência artificial.
   > **Explicação**: O cliente valida se o software atende às suas necessidades reais de negócio.

4. Qual a característica principal do Teste Exploratório?
   - [ ] Seguir um roteiro rígido passo a passo.
   - [x] Aprendizado, design de teste e execução ocorrem simultaneamente, baseados na experiência do testador.
   - [ ] Não documentar nada do que foi feito.
   - [ ] Usar apenas ferramentas automáticas.
   > **Explicação**: É uma abordagem dinâmica que utiliza a inteligência e intuição do QA para encontrar bugs complexos.

5. O "Teste de Sanidade" (Sanity Test) é geralmente realizado:
   - [ ] Antes de começar a programar.
   - [x] Após receber um build com correções de bugs, para verificar se os problemas específicos foram resolvidos.
   - [ ] Somente em sistemas de saúde.
   - [ ] No final do ano.
   > **Explicação**: Verifica a razoabilidade das correções aplicadas.

6. Testes "Alfa" e "Beta" são tipos de:
   - [ ] Testes Unitários.
   - [ ] Testes de Caixa Branca.
   - [x] Testes de Aceitação.
   - [ ] Testes de Performance.
   > **Explicação**: Alfa ocorre internamente na empresa; Beta ocorre com usuários reais em ambiente externo.

7. Por que automatizar testes de regressão é uma boa prática?
   - [ ] Porque robôs não cansam de repetir as mesmas validações a cada novo build.
   - [ ] Porque é mais caro do que contratar pessoas.
   - [ ] Para que os testadores possam ir embora mais cedo.
   - [ ] Não é uma boa prática.
   > **Explicação**: Libera o QA humano para focar em testes exploratórios e cenários mais complexos.

8. O que é um "Caminho Crítico" no contexto de testes?
   - [ ] A rota mais perigosa para chegar ao trabalho.
   - [x] O fluxo principal que o usuário realiza para completar o objetivo do software (ex: comprar um produto).
   - [ ] O código que tem mais bugs.
   - [ ] A senha do administrador.
   > **Explicação**: É o fluxo que, se falhar, impede o uso principal do sistema.

9. Testes funcionais baseiam-se em:
   - [ ] Como o código foi escrito.
   - [x] No comportamento esperado descrito nos requisitos/especificações.
   - [ ] Na velocidade do processador.
   - [ ] No design das cores da tela.
   > **Explicação**: Focam no "o que" o sistema faz.

10. Um teste exploratório deve ser:
    - [ ] Totalmente aleatório e sem propósito.
    - [x] Estruturado através de "Seções de Teste" (Charters) e focado em objetivos.
    - [ ] Feito apenas por estagiários.
    - [ ] Banido de empresas ágeis.
    > **Explicação**: Mesmo sendo livre, o teste exploratório profissional possui foco e objetivos definidos.
