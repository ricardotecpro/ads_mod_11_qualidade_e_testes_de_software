# Quiz 16 - Projeto Integrador Final 🎓

1. O que resume melhor o objetivo do curso de Qualidade e Testes?
   - [ ] Aprender a reclamar do código dos outros.
   - [x] Capacitar para planejar, executar e automatizar testes garantindo a excelência do software.
   - [ ] Tornar-se um hacker profissional.
   - [ ] Aprender apenas a usar o Jira.
   > **Explicação**: O curso visa formar profissionais completos em garantia de qualidade (QA).

2. Na pirâmide de testes ideal, qual o papel dos Testes Manuais?
   - [ ] Devem ser extintos.
   - [x] Focar em Testes Exploratórios, Usabilidade e validações de cenários complexos onde o humano agrega valor.
   - [ ] Focar em repetir os mesmos cliques todo dia.
   - [ ] Somente se a automação falhar.
   > **Explicação**: O humano deve focar no que exige criatividade e percepção, automatizando o repetitivo.

3. Qual a principal lição do TDD para a arquitetura de sistemas?
   - [ ] Escrever código rápido.
   - [x] Que o código deve ser construído para ser testável (desacoplado e modular).
   - [ ] Que não precisamos de arquitetos de software.
   - [ ] Usar Python é sempre melhor.
   > **Explicação**: Testar primeiro obriga a criar uma estrutura limpa.

4. Ao final de um projeto integrador, o que é um "Relatório de Encerramento"?
   - [ ] Uma lista de quem trabalhou no projeto.
   - [x] Um documento com o resumo das atividades, métricas de defeitos e a recomendação de liberação (Go/No-go).
   - [ ] O manual de instalação.
   - [ ] A nota fiscal do serviço.
   > **Explicação**: É o aval final do QA sobre a saúde do produto.

5. A "Cultura de Qualidade" em uma empresa significa que:
   - [ ] Só o QA é responsável pelos bugs.
   - [x] Qualidade é um valor compartilhado por todos, de desenvolvedores a gestores (Whole Team Approach).
   - [ ] A empresa tem muitos certificados na parede.
   - [ ] Não existem bugs em nenhum lugar.
   > **Explicação**: Qualidade nasce no planejamento e segue até o pós-lançamento por todo o time.

6. O uso de "Shift Left Testing" ajuda a:
   - [ ] Aumentar o custo do projeto.
   - [x] Encontrar e prevenir bugs nas fases iniciais (requisitos/design), reduzindo custos drasticamente.
   - [ ] Mudar a cultura para o lado esquerdo.
   - [ ] Diminuir a velocidade de desenvolvimento.
   > **Explicação**: Corrigir um erro no papel é 100x mais barato que no código pronto.

7. Qual ferramenta é essencial para a saúde de longo prazo de um pipeline de CI/CD?
   - [ ] Photoshop.
   - [x] Uma suíte robusta de Testes de Regressão Automatizados.
   - [ ] Um antivírus potente.
   - [ ] Muitos comentários no código.
   > **Explicação**: A automação garante que cada mudança não "quebre" o passado.

8. Em um projeto integrador, a "Rastreabilidade" prova que:
   - [ ] O QA trabalhou 8 horas por dia.
   - [x] Todos os requisitos de negócio foram cobertos por pelo menos um teste.
   - [ ] O banco de dados está em nuvem.
   - [ ] O projeto foi caro.
   > **Explicação**: Vincula o plano de testes às necessidades do cliente.

9. O papel do QA na agilidade (Scrum) é:
   - [ ] Testar tudo somente no último dia da sprint.
   - [x] Participar desde o refinamento das estórias, definindo critérios de aceite e testando continuamente.
   - [ ] Apenas participar da reunião diária.
   - [ ] Escrever o manual do usuário.
   > **Explicação**: O QA ágil é parte integrante e ativa do desenvolvimento diário.

10. Qual o próximo passo para um profissional que concluiu este curso?
    - [ ] Parar de estudar.
    - [x] Continuar praticando, explorar novas ferramentas e aplicar os conceitos de mentalidade de qualidade em projetos reais.
    - [ ] Mudar de profissão.
    - [ ] Deletar os arquivos do curso.
    > **Explicação**: A área de QA exige aprendizado contínuo frente às novas tecnologias e metodologias.
