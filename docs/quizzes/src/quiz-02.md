# Quiz 02 - Modelos de Qualidade 📊

1. O que significa a sigla CMMI?
   - [ ] Control Maturity Model for Integration.
   - [x] Capability Maturity Model Integration.
   - [ ] Code Management Model Integration.
   - [ ] Computer Maturity and Management Institute.
   > **Explicação**: CMMI é o modelo de referência global para maturidade de processos.

2. Qual o nível do CMMI onde os processos são "imprevisíveis e reativos"?
   - [x] Nível 1 - Inicial.
   - [ ] Nível 2 - Gerenciado.
   - [ ] Nível 3 - Definido.
   - [ ] Nível 5 - Em Otimização.
   > **Explicação**: No nível 1, a organização não tem processos padronizados e depende de esforços heróicos individuais.

3. No Nível 5 do CMMI, o foco principal é:
   - [ ] Começar a documentar os requisitos.
   - [ ] Criar um banco de dados SQL.
   - [x] A melhoria contínua dos processos baseada em dados.
   - [ ] Contratar mais desenvolvedores.
   > **Explicação**: O nível máximo foca em otimização e inovação contínua.

4. O modelo MPS.br foi criado para:
   - [ ] Substituir o CMMI em todo o mundo.
   - [x] Melhorar os processos em empresas brasileiras, especialmente PMEs.
   - [ ] Regularizar o uso de software pirata no Brasil.
   - [ ] Apenas certificar empresas multinacionais.
   > **Explicação**: O MPS.br é uma iniciativa brasileira (SOFTEX) para tornar a certificação de qualidade mais acessível.

5. Qual nível do MPS.br corresponde ao "G - Parcialmente Gerenciado"?
   - [x] O nível de entrada (mais básico).
   - [ ] O nível intermediário.
   - [ ] O nível máximo de excelência.
   - [ ] Um nível que não exige documentação.
   > **Explicação**: O nível G é o primeiro passo na jornada de maturidade do MPS.br.

6. O que mede a "Densidade de Defeitos"?
   - [ ] O peso total do executável do software.
   - [x] O número de defeitos encontrados dividido pelo tamanho do código (ex: KLOC).
   - [ ] A velocidade da internet durante os testes.
   - [ ] O número de testadores por projeto.
   > **Explicação**: É uma métrica que indica a concentração relativa de problemas no código.

7. Uma empresa com Nível 2 do CMMI foca em:
   - [ ] Inovação tecnológica radical.
   - [x] Gestão de projetos, garantindo que os processos básicos sejam repetíveis.
   - [ ] Eliminar todos os bugs do mundo.
   - [ ] Mudar de linguagem de programação todo mês.
   > **Explicação**: O foco do nível 2 é estabilizar a gestão para que o sucesso de um projeto possa ser repetido em outros.

8. Qual o objetivo da métrica DRE (Defect Removal Efficiency)?
   - [ ] Medir quantos commits um desenvolvedor faz por dia.
   - [x] Avaliar a eficácia das atividades de teste antes do lançamento.
   - [ ] Contar quantos usuários acessam o sistema.
   - [ ] Medir o tempo de boot do servidor.
   > **Explicação**: A DRE compara defeitos internos vs defeitos externos (produção).

9. Qual nível do CMMI introduz a gestão quantitativa (estatística) dos processos?
   - [ ] Nível 2.
   - [ ] Nível 3.
   - [x] Nível 4.
   - [ ] Nível 1.
   > **Explicação**: No nível 4, a organização usa métricas para controlar o desempenho dos processos estatisticamente.

10. O MPS.br possui quantos níveis de maturidade?
    - [ ] 5 níveis.
    - [x] 7 níveis (A a G).
    - [ ] 3 níveis.
    - [ ] 10 níveis.
    > **Explicação**: O MPS.br desdobra os níveis do CMMI para permitir uma progressão mais gradual (G, F, E, D, C, B, A).
