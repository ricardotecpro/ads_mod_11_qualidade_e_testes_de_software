# Quiz 03 - Ciclo de Vida de Testes (STLC) 🔄

1. O que significa STLC?
   - [ ] Software Technical Level Control.
   - [x] Software Testing Life Cycle.
   - [ ] System Tools Logic Cycle.
   - [ ] Standard Testing Language Code.
   > **Explicação**: STLC é o ciclo de vida dedicado exclusivamente às atividades de teste.

2. Qual a primeira fase do STLC?
   - [ ] Planejamento de Testes.
   - [x] Análise de Requisitos de Teste.
   - [ ] Execução de Testes.
   - [ ] Configuração de Ambiente.
   > **Explicação**: Tudo começa entendendo o que deve ser testado através da análise de requisitos.

3. Em qual fase são criados os scripts e roteiros de teste?
   - [ ] Análise de Requisitos.
   - [ ] Planejamento.
   - [x] Desenvolvimento de Casos de Teste.
   - [ ] Encerramento.
   > **Explicação**: É nesta fase que o QA detalha os passos, dados e resultados esperados.

4. A fase de "Configuração do Ambiente de Teste" envolve:
   - [ ] Escrever o código da aplicação.
   - [x] Preparar o hardware, software e dados necessários para rodar os testes.
   - [ ] Treinar os usuários finais.
   - [ ] Corrigir bugs de interface.
   > **Explicação**: O ambiente deve ser uma réplica (ou o mais próximo possível) do ambiente real/produção.

5. O que é feito na fase de "Execução de Testes"?
   - [ ] Apenas o planejamento do que será feito.
   - [ ] A criação de mocks.
   - [x] Rodar os casos de teste e reportar os defeitos encontrados.
   - [ ] Deletar o banco de dados.
   > **Explicação**: É a fase "mão na massa", onde comparamos o comportamento real com o esperado.

6. Qual o objetivo da fase de "Encerramento de Testes"?
   - [ ] Iniciar um novo projeto.
   - [x] Analisar os resultados, coletar métricas e criar o relatório final de fechamento.
   - [ ] Garantir que nenhum bug novo será criado.
   - [ ] Formatar os computadores do time.
   > **Explicação**: Serve para documentar o que foi alcançado e as lições aprendidas no ciclo.

7. No STLC, quando as métricas de teste são coletadas?
   - [ ] Somente no planejamento.
   - [ ] Somente se houver bugs.
   - [x] Ao longo da execução e formalmente no encerramento.
   - [ ] Nunca se o software for pequeno.
   > **Explicação**: As métricas servem para medir a saúde do projeto e a eficácia dos testes.

8. Qual a relação entre STLC e SDLC?
   - [ ] Eles são exatamente a mesma coisa.
   - [x] O STLC ocorre em paralelo ou integrado às fases do SDLC (Desenvolvimento).
   - [ ] O STLC substitui o SDLC.
   - [ ] Não possuem relação.
   > **Explicação**: O teste deve estar presente em cada etapa do desenvolvimento (Shift Left).

9. Os "Dados de Teste" são preparados em qual fase?
   - [ ] Execução.
   - [x] Desenvolvimento de Casos de Teste / Configuração de Ambiente.
   - [ ] Análise de Requisitos.
   - [ ] Depois que o bug é encontrado.
   > **Explicação**: É preciso ter dados prontos (contas, valores) antes de iniciar a execução.

10. Se um requisito mudar durante a execução, o que o time deve fazer?
    - [ ] Ignorar a mudança e terminar o que começou.
    - [ ] Parar tudo e recomeçar o planejamento do zero.
    - [x] Adaptar os planos e casos de teste afetados através da análise de impacto.
    - [ ] Culpar o desenvolvedor.
    > **Explicação**: O processo de QA deve ser resiliente e adaptável a mudanças de escopo.
