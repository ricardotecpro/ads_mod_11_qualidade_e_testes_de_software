# Quiz 10 - Testes Não Funcionais e Usabilidade 🧠

1. O que caracteriza um Teste Não Funcional?
   - [ ] Verificar se o botão de login redireciona para a home.
   - [x] Avaliar aspectos como desempenho, segurança e usabilidade do sistema.
   - [ ] Somente testes feitos por estagiários.
   - [ ] Testes que não funcionam.
   > **Explicação**: Foca no "como" o sistema executa suas tarefas, não no "quê" ele faz.

2. Qual a diferença entre Teste de Carga e Teste de Estresse?
   - [ ] São sinônimos.
   - [x] Carga testa dentro do limite projetado; Estresse leva o sistema ao ponto de ruptura.
   - [ ] Estresse é para os desenvolvedores; Carga é para os servidores.
   - [ ] Carga testa apenas o banco de dados.
   > **Explicação**: Estresse busca descobrir o limite máximo suportado até que o sistema falhe.

3. Testes de Segurança visam proteger o software contra:
   - [ ] Poeira no servidor.
   - [x] Acesso não autorizado, vazamento de dados e vulnerabilidades como SQL Injection.
   - [ ] Cliques excessivos do usuário.
   - [ ] Quedas de energia.
   > **Explicação**: A segurança garante a integridade e confidencialidade das informações.

4. A Heurística de Nielsen "Visibilidade do status do sistema" recomenda:
   - [x] Manter o usuário sempre informado sobre o que está acontecendo (ex: barra de progresso).
   - [ ] Esconder os erros para não assustar o usuário.
   - [ ] Usar cores muito vibrantes.
   - [ ] Não usar ícones.
   > **Explicação**: O usuário deve saber se o sistema está processando algo ou se terminou uma tarefa.

5. Testes de Usabilidade focam em:
   - [ ] Apenas na beleza das fontes.
   - [x] Facilidade de aprendizado, eficiência de uso e satisfação do usuário.
   - [ ] Qual o preço do software.
   - [ ] Se o código está comentado.
   > **Explicação**: Um software útil deve ser fácil e intuitivo para o seu público-alvo.

6. O que é o tempo de resposta (Latency)?
   - [ ] O tempo que o usuário leva para digitar.
   - [x] O intervalo entre o envio de uma requisição e o recebimento da resposta pelo sistema.
   - [ ] O tamanho do arquivo em bytes.
   - [ ] O número de pixels na tela.
   > **Explicação**: Menor latência significa um sistema mais ágil e melhor percepção de performance.

7. Níveis de contraste e suporte a leitores de tela fazem parte de:
   - [ ] Teste de Stress.
   - [ ] Teste de Fumaça.
   - [x] Teste de Acessibilidade.
   - [ ] Teste de Aceitação.
   > **Explicação**: Acessibilidade garante que o software seja inclusivo para pessoas com deficiência.

8. Qual o principal objetivo do Teste de Portabilidade?
   - [ ] Facilitar o transporte do computador.
   - [x] Verificar se o sistema funciona em diferentes navegadores, sistemas operacionais e dispositivos.
   - [ ] Reduzir o peso do código.
   - [ ] Mudar o software de servidor todo mês.
   > **Explicação**: Garante a compatibilidade multiplataforma.

9. Testes "Não Funcionais" devem ser realizados:
   - [ ] Somente após o lançamento oficial.
   - [x] De forma planejada e integrada ao ciclo de desenvolvimento.
   - [ ] Somente se houver reclamação de lentidão.
   - [ ] Nunca, pois são muito caros.
   > **Explicação**: Performance e segurança devem ser pensadas desde a arquitetura.

10. A métrica "Taxa de Erro de Tarefa" em testes de usabilidade indica:
    - [ ] Quantos bugs de código existem.
    - [x] O percentual de usuários que não conseguiram completar uma ação pretendida.
    - [ ] O número de vezes que o servidor caiu.
    - [ ] A velocidade da conexão.
    > **Explicação**: Ajuda a identificar gargalos no design da interface e do fluxo.
