# Documentação e Controle 📄
## Aula 04 - Artefatos de QA

---

## Por que documentar? ✍️

- Memória do projeto
- Rastreabilidade
- Evidência para o cliente

---

## Plano de Teste 🗺️

- **O que** será testado?
- **Como** será testado?
- **Quando** será entregue?
- **Quem** fará os testes?

---

## Casos de Teste 🧪

### Componentes Essenciais:
1.  ID e Título
2.  Pré-condições
3.  Passos
4.  Dados de Teste
5.  Resultado Esperado

---

## Relatório de Bug 🐛

- Título claro
- Passos para reproduzir
- **Severidade** (Impacto técnico)
- **Prioridade** (Urgência de negócio)
- Evidências (Prints/Logs)
